> 코드를 LLM으로 직접 분석하여 발견한 **버그, 불편사항, 개선사항**을 정리한 보고서입니다.

## 🐛 A. 버그 (Bugs)

### A1. 스펠링 테스트에서 키보드 핸들러 누적 문제
- **파일**: `js/phases/vocab.js` (L578-583)
- **문제**: 스펠링 테스트에서 `document.addEventListener('keydown', handleKeydown)`로 직접 리스너를 등록하면서, 이전 리스너를 `window._spellingKeydownHandler`로 제거하려고 합니다. 그러나 앱에서 다른 화면으로 이동할 때 이 리스너를 **명시적으로 제거하지 않습니다**. 스펠링 테스트를 나간 후에도 `handleKeydown`이 계속 실행되어, 다른 화면에서 a~z 키를 누르면 의도치 않은 동작이 발생할 수 있습니다.
- **심각도**: 🔴 높음
- **증상**: 스펠링 테스트 후 다른 화면에서 키보드 입력이 먹히거나 `preventDefault`가 걸림

### A2. 스펠링 테스트에서 keyboard 매니저와 직접 리스너 충돌
- **파일**: `js/phases/vocab.js` (L546-586)
- **문제**: 스펠링 테스트는 `keyboard.clearShortcuts()`로 매니저를 비우고 자체 `keydown` 리스너를 등록합니다. 그런데 KeyboardManager의 `_onKeydown`은 여전히 동작 중이므로, 두 핸들러가 동시에 이벤트를 처리합니다. 현재는 shortcuts가 비어있어 문제가 안 되지만, 구조적으로 불안정합니다.
- **심각도**: 🟡 중간

### A3. 문장 만들기에서 중복 단어가 있을 때 `used` 판정 오류
- **파일**: `js/phases/vocab.js` (L661)
- **문제**: `selectedWords.includes(w)` 조건으로 `used` 클래스를 붙이는데, 같은 단어가 여러 번 나오는 문장의 경우, 하나만 선택해도 같은 단어의 모든 블록이 used로 표시될 수 있습니다.
- **심각도**: 🟡 중간

### A4. `reading-phases` 뷰에서 sentenceBuild 완료 판정 불일치
- **파일**: `js/app.js` (L104-115), `js/phases/home.js` (L34)
- **문제**: `home.js`에서 섹션 완료 판정은 `phases.vocab && phases.reading && phases.quiz`로 하는 반면, phase picker에서는 `sentenceBuild`도 표시합니다. `sentenceBuild`를 안 해도 ✅ 표시됩니다.
- **심각도**: 🟡 중간

### A5. 리뷰 모드에서 correctCount 계산 논리 오류
- **파일**: `js/phases/quiz.js` (L128-132)
- **문제**: 리뷰 모드에서 3번 틀리면 `correctCount++`로 자동 정답 처리됩니다. 3번 틀린 문제가 "맞힌 문제"로 카운트되어 score가 부풀려집니다.
- **심각도**: 🟡 중간

### A6. 성적표(Report)가 전체 학생의 별(stars)을 표시
- **파일**: `js/admin/report.js` (L33)
- **문제**: `store.getStars()`는 **전역** 별 카운트입니다. 학생별 별 카운트가 아니라 앱 전체의 별을 표시합니다.
- **심각도**: 🔴 높음 (다중 학생 사용 시)

### A7. 관리자 비밀번호가 화면에 노출
- **파일**: `js/admin/admin.js` (L240)
- **문제**: 로그인 화면에 "기본 비밀번호: 1234"가 그대로 표시됩니다. 학생이 이 화면을 보면 관리자 페이지에 접근할 수 있습니다.
- **심각도**: 🟡 중간

## 😤 B. 사용성 불편사항 (UX Issues)

### B1. 읽기 완료 버튼을 누르면 확인 없이 즉시 완료 처리
- **파일**: `js/phases/reading.js` (L122-131)
- **문제**: "읽기 완료!" 버튼을 한 번 클릭하면 바로 완료 처리되고 자동 이동됩니다. 실수로 눌렀을 때 취소할 수 없습니다.

### B2. 전체 읽기 도중 멈출 수 없음
- **파일**: `js/phases/reading.js` (L92-106)
- **문제**: 전체 읽기 중단을 위한 "멈추기" 버튼이 없습니다. 긴 본문에서 시작하면 끝까지 들어야 합니다.

### B3. 학습 단계 순서를 강제하지 않음
- **문제**: 단어 학습을 건너뛰고 바로 문제 풀기에 들어갈 수 있습니다.

### B4. 뒤로 가기 시 진행 상태 소실
- **문제**: 단어 카드 진행 중 뒤로 가면 다시 1번부터 시작합니다.

### B5. 관리자 세션이 새로고침으로 초기화됨
- **파일**: `js/admin/admin.js` (L10)
- **문제**: `window._adminLoggedIn`은 메모리 변수이므로 새로고침 시 다시 로그인 필요

### B6. "따라 말하기" 모드에서 실제 녹음/비교 기능 없음
- **파일**: `js/phases/reading.js` (L241-263)
- **문제**: 3초 카운트다운만 하고 실제 마이크를 사용하지 않습니다.

### B7. sentenceBuilding 데이터가 각 reading에 2문장뿐
- **문제**: 본문에 14~16문장이 있는 것에 비해 연습량이 매우 적습니다.
